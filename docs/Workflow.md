# Workflow

- Nhanh: main, develop, feat/<scope>, fix/<scope>. Moi nhanh phat trien tu develop, PR ve develop.
- Commit: Conventional Commits (feat/fix/chore/docs/test). Giu PR < 400 dong, kem screenshot/clip, khong secret.
- Review: tu chay emulator + seed truoc khi mo PR; dinh kem ket qua flutter analyze.
- Phan cong: A1 cloud (feat/cloud-core-social), A2 UI (feat/ui-social-feed), A3 search (feat/search-unified), A4 DevOps/QA (chore/emulator-ci-templates).
- Nhan task: pull develop -> tao branch -> code + test -> cap nhat docs neu can -> mo PR -> check GitHub Actions -> resolve review -> merge.
